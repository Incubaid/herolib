What is JSON-RPC?

JSON-RPC is a stateless, light-weight remote procedure call (RPC) protocol. It defines several data structures and the rules around their processing. It is transport agnostic in that the concepts can be used within the same process, over sockets, over HTTP, or in many various message passing environments.

Why JSON-RPC 2.0? Unlike REST APIs, JSON-RPC provides a simpler, more predictable interface with built-in error handling, batch requests, and a standardized specification that ensures consistency across implementations.

Getting Started

HeroServer provides a simple and intuitive API interface. All requests are sent as HTTP POST requests to the server endpoint with a JSON-RPC 2.0 formatted payload.

Basic Request Structure:
{
  "jsonrpc": "2.0",
  "method": "method_name",
  "params": { /* your parameters */ },
  "id": 1
}
Quick Start Steps:
  • Explore the API: Browse the available endpoints below to see what methods are available
  • Read the Documentation: Click on "API Documentation" in the navbar to view detailed API specifications
  • Test with curl: Use the provided curl examples to test API calls directly
  • Integrate: Use any HTTP client library in your preferred programming language

Quick Start

Getting Handler Information

Get detailed information about a specific API handler in JSON format:

curl ${server_info.base_url}/json/[handler_name]
Making API Calls

All API endpoints use JSON-RPC 2.0 format. Here's a basic example:

curl -X POST ${server_info.base_url}/api/[handler_name] \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "method_name",
    "params": {
      "param1": "value1",
      "param2": "value2"
    },
    "id": 1
  }'
Tip: Visit the documentation pages for specific examples with copy-to-clipboard functionality
